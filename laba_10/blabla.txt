1) Что такое JSP и ее состав и назначение?
JavaServer Pages (JSP) - это технология, используемая для создания динамических веб-страниц на основе Java и HTML. Основными компонентами JSP являются HTML-код, который отображается в браузере, и Java код, который выполняется на сервере. JSP предоставляет удобный способ встраивания Java кода в HTML, что позволяет создавать динамические страницы, которые могут адаптироваться к различным ситуациям и пользователям.
2) Жизненный цикл JSP и отличие JSP и Servlet:
Жизненный цикл JSP включает несколько этапов, таких как компиляция JSP в сервлет, инициализация, выполнение и уничтожение. Основное отличие между JSP и сервлетами заключается в том, что JSP является HTML-страницей с внедренным Java кодом, тогда как сервлет является Java классом, который генерирует содержимое веб-страницы.
3) Теги JSP и их назначение:
Некоторые основные теги JSP включают <% %> для вставки Java кода, <%= %> для вывода значения переменной или выражения, <%@ %> для директив, <%! %> для объявления методов и переменных класса, <%-- --%> для комментариев, <jsp:include> для включения другого ресурса и <jsp:forward> для перенаправления запроса на другую страницу.
4) Неявные объекты JSP:
Неявные объекты JSP - это объекты, которые автоматически доступны в JSP странице без необходимости их объявления. Некоторые из них включают объекты request, response, session, application, out и exception.
5) Области видимости переменных JSP:
Переменные в JSP могут иметь три области видимости: page, request и session. Переменные с областью видимости page доступны только в текущей JSP странице, переменные с областью видимости request доступны в рамках текущего запроса, а переменные с областью видимости session доступны на протяжении всей сессии пользователя.
6) Что такое PageContext:
PageContext - это объект, предоставляемый контейнером сервлетов для каждой JSP страницы. Он предоставляет доступ к различным сервлет-специфическим объектам, таким как request, response, session, application и другие.
7) Что такое EL и как он используется:
EL (Expression Language) - это язык выражений, используемый в JSP для упрощения доступа к данным в JavaBean, объектам сессии и другим областям. Он позволяет встраивать выражения прямо в код JSP для удобства доступа к данным и выполнения операций.
8) Как задать и настроить страницу ошибок (error page):
Страница ошибок может быть задана в директиве <%@ page %> в начале JSP страницы с использованием атрибута errorPage. Этот атрибут указывает на другую JSP страницу, которая будет отображаться в случае возникновения ошибки на текущей странице. Дополнительно, в web.xml можно настроить глобальные страницы ошибок для различных HTTP кодов ошибок.
9) Взаимодействие JSP-Servlet-JSP:
JSP и сервлеты часто взаимодействуют между собой в веб-приложениях. Сервлеты могут обрабатывать запросы от клиента, выполнять бизнес-логику и передавать данные на JSP страницы для отображения. JSP страницы, в свою очередь, могут содержать HTML для отображения данных и встроенный Java код для обработки логики представления. Таким образом, сервлеты и JSP взаимодействуют для обработки запросов и формирования ответов в веб-приложениях.
